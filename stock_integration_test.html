Stock Model Methods

Available Methods:
  • getCurrentStock($productId) - Get current stock level
  • getAvailableStock($productId) - Get available stock
  • getItemStock($itemId, $warehouseId) - Get detailed stock info
  • addStock($data) - Add stock (purchase/production)
  • removeStock($itemId, $quantity) - Remove stock (sales)
  • reserveStock($itemId, $quantity) - Reserve stock for orders
  • transferStock($from, $to, $quantity) - Transfer between warehouses
Stock Status Types:
  • Available - Ready for sale
  • Reserved - Reserved for orders
  • Blocked - Quality issues
  • Quarantine - Under inspection
  • Expired - Past expiry date
  • Damaged - Damaged goods

Stock Integration Flow

Complete Stock Integration Workflow:

  2. Sales Order Creation → Validate stock availability
  3. Order Confirmation → Reserve stock for the order
  4. Dispatch Preparation → Check reserved stock
  5. Dispatch Note → Deduct stock from inventory
  6. Invoice Generation → Confirm stock deduction
  7. Payment Collection → Update financial records
  8. Returns Processing → Add stock back to inventory

Stock Integration API Endpoints

Available API Endpoints:
  • GET /api/stock/product/{id} - Get product stock info
  • POST /api/stock/check-availability - Check multiple products
  • GET /api/stock/warehouse/{id} - Get warehouse stock
  • POST /api/stock/reserve - Reserve stock for order
  • POST /api/stock/release - Release reserved stock
